NETWORK DEVICES
WHAT'S A NETWORK?
A COMPUTER NETWORK IS A DIGITAL TELECOMMUNICATION NETWORK WHICH ALLOWS NODES TO SHARE RESOURCES.

A CLIENT IS A DEVICE THAT ACCESSES A SERVICE MADE AVAILABLE BY A SERVER.
A SERVER IS A DEVICE THAT PROVIDES FUNCTION OR SERVICES FOR CLIENTS.



INTERFACES AND CABLES
ETHERNET IS A COLLECTION OF NETWORK PROTOCOLS/STANDARDS

BITS AND BYTES
I KILOBIT (KB) - 1,000 BITS
1 MEGABIT (MB) - 1,000,000, BITS
1 GIGABIT (GB) - 1,000,000,000 BITS
1 TERABIT (TB) - 1,000,000,000,000 BITS

UTP CABLES (10BASE-T AND 100BASE-T) - USES TWO PAIRS (4 WIRES)
UTP CABLES (1000BASE-T AND 10GBASE-T) - USES 4 PAIRS (8 WIRES)

TRANSMIT WITH 1 - 2 AND RECEIVE WITH 3 AND 6 - FOR PC, ROUTER AND OTHER DEVICES EXCEPT SWITCHES USING STRAIGHT THROUGH CABLE.
SWITCHES TRANSMIT FROM 3 - 6 AND RECEIVE WITH 1 -2

STRAIGHT CABLES ARE USE TO CONNECT OPPOSITE DEVICES ON THE NETWORK.

CROSSOVER CABLES ARE USE TO CONNECT SAME TYPES OF DEVICES ON THE NETWORK

PROTOCOLS AND STANDARDS
A PROTOCOLS ARE SET OF RULES DEFINING HOW DATA SHOULD BE COMMUNICATED BETWEEN DEVICES OVER A NETWORK.

A STANDARD IS AN AGREED-UPON SPECIFICATION THAT DESCRIBE HOW A PROTOCOL OR TECHNOLOGY SHOULD WORK.

THE TCP/IP MODEL
APPLICATION LAYER - PROTOCOL FOR COMMUNICATION BETWEEN APPLICATION PROCESS, CREATE AND INTERPRET DATA.

TRANSPORT LAYER - PROVIDES END-TO-END COMMUNICATION BETWEEN APPLICATION PROCESSES USING PORT NUMBERS.

INTERNET LAYER - PROVIDES END-TO-END COMMUNICATION BETWEEN HOSTS ACROSS NETWORKS USING IP ADDRESSES AND ROUTERS.

LOCAL NETWORK LAYER - PROVIDES HOP-TO-HOP DELIVERY WITHIN A LOCAL USING MAC ADDRESSES AND SWITCHES.

PHYSICAL LAYER - SENDS BITS AS ELECTRICAL, OPTICAL, OR RADIO SIGNALS OVER THE PHYSICAL MEDIUM.

PROTOCOL DATA UNITS
THE COMBINATION OF DATA AND L4 HEADER IS CALLED A SEGMENT (TCP) OR DATAGRAM (UDP).
TCP CREATES SEGMENTS AND UDP CREATES DATAGRAMS.

A SEGMENT OR DATAGRAM IS A LAYER 4 PDU (L4PDU)
A PACKET IS A LAYER 3 PDU (L3PDU)
A FRAME IS A LAYER 2 PDU (L2PDU)
