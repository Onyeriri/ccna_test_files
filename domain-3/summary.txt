A DEFAULT GATEWAY AND DEFAULT ROUTER ARE SYNONYMS.

5 STEPS IT TAKES FOR A ROUTER TO FORWARD RECEIVED DATA FRAME.
-1A - THE FRAME HAS NO ERRORS
-1B - THE FRAME DESTINATION DATA-LINK IS THE ROUTERS ADDRESS
-2 - DE-ENCAPSULATE THE PACKET FROM DATA-LINK
-3 - MAKE A ROUTING DECISION
-4 - ENCAPSULATE THE PACKET
-5 - TRANSMIT THE FRAME

MATCHING DECISION
-SUBNET ID
-MASK

FORWARDING DECISION
-NEXT ROUTER
-OUT INTERFACE

PREFIX MEANS SUBNET ID

CONFIGURING STATIC ROUTES
A STATIC NETWORK ROUTE DEFINES EITHER A SUBNET OR AN ENTIRE CLASS A, B OR C NETWORK IN THE IP ROUTE COMMAND.

-A DEFAULT ROUTE DEFINES THE SET OF ALL DESTINATION IP ADDRESS

-A HOST ROUTE DEFINES A SINGLE IP ADDRESS

USING LOCAL INTERFACE AS OUT GOING FORWARDING METHOD FORCES THE USE OF PROXY ARP

TROUBLESHOOTING STATIC ROUTE
-THE ROUTE IS IN THE ROUTING TABLE BUT IS INCORRECT
-THE ROUTE IS NOT IN THE ROUTING TABLE
-THE ROUTE IS IN THE ROUTING TABLE AND IS CORRECT, BUT THE PACKETS DO NOT ARRIVE AT THE DESTINATION HOST

USING PERMANENT KEYWORD INSTRUCT THE ROUTER TO INCLUDE ROUTE NO MATTER WHAT.

VERIFYING ROAS
BEYOND USING SHOW RUNNING CONFIG COMMAND, ROAS CAN BE BEST VERIFIED WITH TWO COMMANDS:
-SHOW IP ROUTE CONNECTED
-SHOW VLANS

IF YOU UNPLUG THE PHYSICAL INTERFACE THE SUB-INTERFACES STATUS SHOW DOWN/DOWN AND IF YOU SHUTDOWN THE PHYSICAL INTERFACE THE STATUS SHOWS ADMINISTRATIVELY DOWN.

TROUBLESHOOTING ROAS
THE BIGGEST CHALLENGE WHEN TROUBLESHOOTING ROAS HAS TO DO WITH THE FACT THAT IF YOU MIS-CONFIGURE THE ROUTER OR THE SWITCH THERE IS NO WAY OF FIGURING OUT WHERE THE PROBLEM STEMS FROM. THE BEST THING TO DO IS CHECK THE CONFIGURATION REQUIREMENT FOR THE ROAS IN BOTH NETWORK DEVICES.

VLAN ROUTING WITH LAYER 3 SWITCH SVI
IN THIS METHOD THE SWITCH WILL CREATE VIRTUAL INTERFACE FOR ALL THE VLAN CREATED ON THE LAYER 2 WITH AN IP ADDRESS AS THEIR DEFAULT GATEWAY FOR ROUTING OUTSIDE THEIR SUBNET.

FOR ROUTING TO BE EFFECTIVE IN LAYER 3 SWITCH USE THE COMMAND IP ROUTING IN THE GLOBAL CONFIGURATION COMMAND.

TROUBLESHOOTING ROUTING WITH SVI
-SVI INTERFACE STATE WITH AUTO-STATE ENABLED
-MAKE SURE THE VLAN EXIST
-MAKE SURE VLAN HAS ONE INTERFACE CONNECTED AND ACTIVE
-VLAN IS NOT SHUTDOWN OR DELETED
-VLAN INTERFACE IS NOT SHUTDOWN

SVI INTERFACE STATE WITH AUTO-STATE DISABLED
